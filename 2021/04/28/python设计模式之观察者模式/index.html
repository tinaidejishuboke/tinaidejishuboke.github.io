# 观察者模式

观察者模式也叫发布订阅模式，** 定义了对象之间一对多依赖，当一个对象改变状态时，这个对象的所有依赖者都会收到通知并按照自己的方式进行更新。** 举个现实生活的例子，职员们趁老板不在，都在玩着自己的东西，同时观察着前台小姐姐，前台小姐姐在老板回来的时候，发布通知让各同事回到工作状态。

# 四。观察者模式的优缺点

优点：

  • 它使彼此交互的对象保持松耦合
  • 当需要添加其他对象时，无需对主题和观察者做任何修改
  • 可以随时添加和删除观察者

缺点：

  • 实现不当可能会增加复杂性，导致性能降低
  • 通知有时是不可靠的，并导致竞争条件或不一致性
